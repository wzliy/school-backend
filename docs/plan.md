# 高校官网轻量化建设项目计划

## 1. 计划基线

本项目按高校官网轻量化建设方式实施，建设一套可独立部署、便于运营和维护的高校官网系统。项目采用“一个官网主站 + 一个招生就业专题站 + 一套 CMS 后台 + 一个后端服务”的架构，不建设复杂站群平台。

| 项目项 | 计划基线 |
| --- | --- |
| 建设模式 | 轻量化高校官网，前后端分离，单体模块化后端 |
| 标准方案 | 42.8 万元整包 |
| 精简方案 | 约 38.8 万元，压缩页面定制和非核心管理能力 |
| 建设周期 | 16 个自然周，需求与素材按计划确认的前提下 |
| 维护周期 | 项目上线验收后 3 年 |
| 核心交付 | 官网主站、招生就业专题站、CMS 后台、后端 API、部署与项目文档 |
| 技术基线 | Java 25、Spring Boot 4.1.0、MySQL 8、Vue 3、Nginx |

报价、工期和范围以最终合同及双方确认的需求清单为准。云服务器、域名购买、商业证书、短信、邮件、第三方商业软件和等保测评等外部费用不计入本项目软件建设报价。

## 2. 建设范围

### 2.1 官网主站

官网主站提供高校信息发布和公众浏览能力，支持 PC、平板和手机响应式访问。

交付范围：

1. 主站首页，包括导航、Banner、新闻动态、通知公告、专题入口、公共服务入口和友情链接。
2. 学校概况、机构设置、教育教学、学生工作和公共服务等栏目展示。
3. 栏目列表页、内容详情页、单页内容、附件下载和站内搜索。
4. 动态导航、栏目排序、内容置顶与推荐。
5. 基础 SEO、网站地图、robots 配置和页面分享信息。
6. 统一视觉设计及主流终端响应式适配。
7. 使用官网首页、文章列表、文章详情、单页图文、机构展示、公共服务和搜索结果等固定模板呈现内容。

### 2.2 招生就业专题站

建设一个招生就业专题站，使用独立访问入口和专题视觉风格，与主站共用 CMS、用户权限、媒体库和后端服务。

交付范围：

1. 专题站首页和专题 Banner。
2. 招生信息、就业信息、政策公告和校企合作等栏目。
3. 专题内容列表页、详情页和附件下载。
4. 专题栏目、内容和 Banner 的后台维护能力。
5. PC 与移动端响应式适配。
6. 使用招生就业专题首页模板和受控页面区块配置，不单独建设 CMS 或自由页面装修器。

专题站不独立建设一套后台，不形成站群管理平台。招生和就业在同一个专题站内按栏目组织。

### 2.3 CMS 后台

CMS 后台面向学校网站管理员和内容编辑人员，交付以下能力：

1. 登录、退出、密码修改和当前用户信息。
2. 用户、角色、菜单和按钮权限管理。
3. 主站与专题站栏目树管理。
4. 内容新建、编辑、草稿、发布、下线、置顶、推荐和附件管理。
5. Banner 管理，包括位置、图片、链接、排序和生效时间。
6. 媒体库管理，包括图片、文档上传、检索和引用。
7. 站点配置、友情链接和基础 SEO 配置。
8. 登录日志和关键操作日志。
9. 固定页面模板注册表、栏目模板选择和按模板展示编辑字段。
10. 主站首页与招生就业专题首页的预定义区块配置，包括数据栏目、数量、预设样式、排序和启停。

首版采用简化发布流程，不建设复杂多级审核和跨部门工作流。

页面模板采用代码维护的固定注册表，不建设动态模板定义、自由拖拽、任意组件嵌套或管理员上传模板能力。在线报名和咨询留言仅保留外部链接入口，不建设数据采集表单及后台处理流程。

### 2.4 接口、测试与上线

1. 完成官网主站、专题站、CMS 后台与后端 API 的接口联调。
2. 提供 OpenAPI 接口说明和统一错误码。
3. 完成功能、权限、兼容性、响应式和基础安全测试。
4. 提供 MySQL 初始化脚本、部署脚本、Nginx 示例配置和部署说明。
5. 完成约定生产环境部署、域名解析与绑定、HTTPS 配置协助和上线验证。
6. 提供管理员操作说明、测试报告和上线交接清单。

域名注册、服务器采购、ICP备案、商业证书购买和第三方平台审批由校方负责，项目组提供技术配合。

### 2.5 三年维护服务

维护期自上线验收之日起计算，包含：

1. 已交付功能缺陷修复。
2. 网站运行巡检、日志排查和故障定位。
3. 常规依赖安全更新和兼容性修复。
4. 数据库备份与恢复指导。
5. 域名、证书续期和服务器迁移的技术协助。
6. CMS 使用咨询和管理员远程支持。

维护不包含新增业务模块、整体改版、日常内容代运营、批量历史数据整理、驻场服务和第三方资源费用。超出范围的需求按变更流程单独评估。

## 3. 明确不建设的内容

以下内容不在 42.8 万元和 38.8 万元方案内，也不作为当前架构的默认扩展任务：

1. 复杂站群、院系多站点自助建站和多租户管理。
2. 多语言内容、人工翻译和多语种 SEO。
3. 短信、邮件、支付及相关第三方服务接入。
4. 复杂 BI、领导驾驶舱、用户画像和数据大屏。
5. 深度智慧校园集成，包括统一身份认证、数据中台、教务、学工、财务和门禁等业务系统对接。
6. 在线报名、在线缴费、咨询留言和招聘撮合等复杂互动业务。
7. 复杂多级审核、跨部门流程和电子签章。
8. 商业全文检索、视频转码、直播和大规模音视频分发。
9. 旧站全量数据迁移、历史数据清洗和内容代录入。
10. 等保测评、渗透测试认证和专项安全整改服务。

新增上述需求时，须形成变更单，重新评估费用、工期、架构和验收标准。

## 4. 技术架构

### 4.1 架构原则

1. 采用轻量化单体模块架构，一个 Java 服务承载后台 API 和公开 API。
2. 主站与专题站共用栏目、内容、媒体和权限模型，通过 `site_type` 区分数据。
3. 不引入微服务、服务注册中心、消息队列和分布式事务。
4. MySQL 作为必选持久化组件；Redis 仅在缓存需求明确时启用。
5. 文件存储默认使用服务器本地目录，可按部署环境切换 MinIO。
6. 前端静态资源和文件访问由 Nginx 统一代理。
7. 页面采用固定模板库和预定义区块，模板配置只保存经过白名单校验的结构化数据。
8. 栏目、内容和页面区块共用站点隔离规则，禁止主站与专题站跨站点引用数据源。

### 4.2 技术基线

| 层级 | 技术选型 |
| --- | --- |
| 后端语言 | Java 25 |
| 构建工具 | Gradle 9.x，编译目标与运行时均为 Java 25 |
| 后端框架 | Spring Boot 4.1.0 |
| 安全框架 | Spring Security OAuth2 Resource Server + Spring JwtEncoder/JwtDecoder + BCrypt |
| 数据访问 | MyBatis Spring Boot Starter 4.0.1 / MyBatis |
| 数据库 | MySQL 8.x |
| 缓存 | Redis，可选启用 |
| 文件存储 | 本地文件存储，保留 MinIO 适配接口 |
| API 文档 | Springdoc OpenAPI 3.0.3 / Swagger UI |
| 后台前端 | Vue 3 + TypeScript + Vite + Element Plus |
| 官网前端 | Vue 3 + TypeScript + Vite |
| 部署 | Nginx + Java 25 服务 + MySQL 8 |

### 4.3 部署结构

```text
访问用户
   |
Nginx
├── /             官网主站与招生就业专题站静态资源
├── /admin        CMS 后台静态资源
├── /api          school-backend
└── /uploads      媒体文件

school-backend (Java 25)
├── Admin API
├── Portal API
├── Auth & Security
├── CMS Modules
└── Storage Adapter
   |
MySQL 8
└── Redis / MinIO（按部署需要启用）
```

## 5. 里程碑与工期

项目计划周期为 16 个自然周。视觉确认、素材提供、备案和域名审批延迟造成的等待时间不计入开发工期。

| 里程碑 | 周期 | 主要工作 | 阶段交付与验收门槛 |
| --- | --- | --- | --- |
| M0 项目启动 | 第 1 周 | 需求确认、内容盘点、责任人和环境确认 | 范围清单、内容清单、项目排期确认 |
| M1 设计与架构 | 第 2-3 周 | 信息架构、八类模板矩阵、首页与内页视觉、专题风格、技术设计 | 模板字段、原型与视觉稿确认，系统设计评审通过 |
| M2 后端基础 | 第 3-5 周 | Java 25 工程、数据库、认证、权限和公共能力 | 服务可启动，管理员可登录，权限生效 |
| M3 CMS 核心 | 第 5-8 周 | 栏目、内容、模板注册、页面区块、Banner、媒体库、SEO、配置和日志 | 管理员可按模板完成主要内容运营流程 |
| M4 官网主站 | 第 6-10 周 | 主站七类模板、响应式适配和公开 API | 首页、列表、详情、单页、机构、服务和搜索模板动态取数 |
| M5 专题站 | 第 9-11 周 | 招生就业专题首页模板、专题栏目和内容管理 | 专题模板独立访问并复用 CMS 与页面区块能力 |
| M6 联调与 SEO | 第 11-13 周 | 模板联调、搜索、SEO、真实内容试录 | 八类模板前后台闭环，SEO 回退与空状态生效 |
| M7 测试与验收 | 第 14-15 周 | 功能、权限、兼容性、安全和用户验收测试 | 阻断问题清零，测试报告和验收问题单闭环 |
| M8 上线交付 | 第 16 周 | 部署、域名绑定、上线验证、培训与交接 | 生产环境可访问，交付资料齐全，进入维护期 |

M2 至 M5 可在接口和视觉基线确认后并行推进，以 16 周整体周期为控制目标。

## 6. 工作分解

### 6.1 需求与设计

1. 确认主站与专题站栏目结构、内容来源和管理员角色。
2. 确认主站首页、通用列表、详情、单页和搜索页原型。
3. 确认招生就业专题站首页、列表和详情页原型。
4. 确认 PC 与移动端视觉稿、交互规范和组件规范。
5. 冻结八类模板编码、适用栏目、字段矩阵、区块类型和标准/精简方案差异。
6. 形成需求基线，范围外事项进入变更清单。

### 6.2 后端与数据库

1. 建立 Java 25 + Spring Boot 4.1.0 工程和环境配置。
2. 完成统一响应、异常、参数校验、分页和 OpenAPI。
3. 完成 MySQL 表、初始化数据和索引。
4. 使用 Spring Security OAuth2 Resource Server 和 Spring JwtEncoder/JwtDecoder 完成 JWT 登录、角色权限和操作审计。
5. 完成栏目、内容、附件、Banner、媒体、站点配置、友情链接和 SEO API。
6. 完成官网公开 API、搜索和浏览量能力。
7. 完成存储适配、上传校验和文件访问控制。
8. 增加栏目详情模板与模板配置、内容受控扩展数据和 `cms_page_section` 增量 SQL。
9. 完成模板注册表、编辑 schema、区块管理和公开页面聚合 API，并校验跨站点引用和 JSON 配置。

### 6.3 CMS 后台

1. 建立后台布局、登录、菜单权限和统一请求处理。
2. 完成用户、角色和权限页面。
3. 完成栏目树、内容编辑发布和附件管理页面。
4. 完成 Banner、媒体库、站点配置、友情链接和 SEO 页面。
5. 完成登录日志、操作日志和基础仪表信息。
6. 栏目编辑支持模板选择，内容编辑页根据模板 schema 显示公共字段和扩展字段。
7. 完成主站首页、专题首页的预定义区块管理，不实现自由拖拽装修。

### 6.4 官网与专题站

1. 完成 `HOME`、`ARTICLE_LIST`、`ARTICLE_DETAIL` 和 `SINGLE_PAGE` 模板。
2. 完成 `ORGANIZATION`、`SERVICE_DIRECTORY` 和 `SEARCH_RESULT` 模板。
3. 完成 `RECRUIT_HOME` 模板，并复用文章列表与详情模板。
4. 完成附件下载、友情链接、动态栏目、空状态、异常页和响应式适配。
5. 完成页面标题、关键词、描述回退、canonical、语义结构和网站地图。

### 6.5 测试与交付

1. 执行后端单元测试和核心接口测试。
2. 执行 CMS 内容运营流程与权限边界测试。
3. 执行主站、专题站的浏览器兼容和移动端测试。
4. 执行上传限制、XSS 防护、未授权访问和日志测试。
5. 按模板验收字段显隐、数据来源、区块排序、跨站点拦截、SEO、空状态和桌面/移动端展示。
6. 完成部署演练、数据库备份、域名绑定和回滚验证。
7. 提交源代码、初始化及增量 SQL、模板矩阵、部署文档、接口文档、测试报告和操作说明。

## 8. 验收标准

### 8.1 官网主站

1. 主站首页、栏目列表、内容详情、单页和搜索页可正常访问。
2. 导航、Banner、内容和友情链接均可由 CMS 动态维护。
3. PC、平板和主流手机分辨率下无明显错位、遮挡或横向溢出。
4. 在双方约定测试环境和网络条件下，主要页面达到可接受的加载体验。

### 8.2 招生就业专题站

1. 专题站通过独立入口访问，具有约定的专题视觉。
2. 招生、就业、政策公告等栏目可由 CMS 独立维护。
3. 专题 Banner、列表、详情和附件下载可正常使用。

### 8.3 CMS 后台

1. 管理员可登录、退出并按角色访问菜单和操作。
2. 可维护用户、角色、栏目、内容、Banner、媒体、站点配置和 SEO。
3. 内容发布、下线、置顶、推荐和附件绑定流程正常。
4. 未登录访问受保护接口返回 401，无权限访问返回 403。
5. 关键后台写操作可通过日志追溯。

### 8.4 页面模板

1. 本期八类模板均有模板编码、适用场景、字段矩阵、桌面原型和移动端规则。
2. 栏目只能选择兼容的固定模板，内容编辑页只展示该模板需要的公共字段和扩展字段。
3. 主站首页和专题首页区块可维护数据栏目、数量、预设样式、排序和启停，不能配置任意脚本或组件。
4. 页面区块不得跨 `MAIN_SITE`、`RECRUIT_SITE` 引用栏目；数据源被禁用或删除时有明确拦截或提示。

## 11. 主要风险

| 风险 | 影响 | 应对策略 |
| --- | --- | --- |
| 范围持续扩大 | 报价和工期失控 | 以本计划和确认原型为基线，范围外需求走变更单 |
| 素材和栏目确认延迟 | 前端和内容联调延期 | M0 建立内容责任清单，按里程碑冻结输入 |
| SEO 期望高于 SPA 能力 | 收录效果不确定 | 完成基础 SEO；需要 SSR 或静态化时单独评估 |
| 上传内容存在安全风险 | XSS 或恶意文件 | 富文本过滤、扩展名与大小限制、非执行目录存储 |
| 本地文件不利于迁移 | 环境切换成本增加 | 使用 StorageService 隔离存储实现，保留 MinIO 适配 |
| 外部审批影响上线 | 域名或 HTTPS 无法按期启用 | 提前准备域名、备案、证书和服务器权限 |
| 模板数量持续增加 | 前后端分支和测试量失控 | M1 冻结八类模板，新增结构走变更单 |
| 区块配置演变为页面装修 | 开发复杂度和操作成本上升 | 只开放预定义区块及受控属性，不支持拖拽嵌套和组件上传 |
| JSON 扩展字段失控 | 数据不可检索、不可校验 | 只保存少量模板专属字段，按模板 schema 校验并保留公共字段列 |
| 模板数据源悬空或跨站点 | 页面空白或数据串站 | 保存时校验引用，栏目禁用删除前检查依赖，公开 API 再做站点过滤 |
