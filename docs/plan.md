# 高校官网后端接口建设项目计划

## 1. 计划基线

本项目调整为高校官网后端接口专项建设，只开发和交付 `school-backend`。后端同时服务一个官网主站、一个招生就业专题站和一套 CMS 管理端，但官网页面、专题站页面和 CMS 管理页面均由其他前端团队开发。

| 项目项 | 计划基线 |
| --- | --- |
| 建设模式 | 前后端分离，单体模块化后端，Admin API 与 Portal API 共用一个服务 |
| 开发责任 | 仅负责后端接口、数据库、文件存储、安全、测试、部署和接口文档 |
| 前端责任 | 官网、专题站和 CMS 管理端页面由其他团队设计、开发、测试和发布 |
| 建设周期 | 后端完整范围基线 10 个自然周，已完成工作计入对应里程碑 |
| 维护周期 | 后端验收后 3 年，仅覆盖本项目后端交付物 |
| 核心交付 | Java 源码、Admin API、Portal API、SQL、OpenAPI、测试与部署文档 |
| 技术基线 | Java 25、Spring Boot 4.1.0、MySQL 8、Nginx API 反向代理 |
| 商务基线 | 原 42.8/38.8 万元前后端整包报价不再适用，后端范围单独核价 |

报价、工期和范围以最终合同及双方确认的后端接口清单为准。云服务器、域名、商业证书、第三方服务、前端开发和前端维护费用不计入本后端专项范围。

## 2. 建设范围

### 2.1 Admin API

Admin API 面向外部 CMS 管理端，交付以下后端能力：

1. 登录、退出、密码修改和当前用户信息。
2. 用户、角色、菜单和按钮权限管理。
3. 主站与专题站栏目树管理。
4. 内容新建、编辑、草稿、发布、下线、置顶、推荐和附件管理。
5. Banner 管理，包括位置、图片、链接、排序和生效时间。
6. 媒体库管理，包括图片、文档上传、检索和引用。
7. 站点配置、友情链接和基础 SEO 配置。
8. 登录日志和关键操作日志。
9. 固定页面模板注册表、栏目模板选择和按模板展示编辑字段。
10. 主站首页与招生就业专题首页的预定义区块配置，包括数据栏目、数量、预设样式、排序和启停。

Admin API 使用 JWT 鉴权和权限编码控制访问。首版采用简化发布流程，不建设复杂多级审核和跨部门工作流。

### 2.2 Portal API

Portal API 面向外部官网和专题站前端，交付以下公开数据能力：

1. 站点配置、导航、Banner 和友情链接查询。
2. 栏目树、栏目详情、栏目内容分页和内容详情查询。
3. 主站首页与招生就业专题首页的页面区块聚合数据。
4. 站内搜索、附件下载信息和浏览量更新。
5. SEO 标题、关键词、描述、canonical 所需数据及回退结果。
6. 只返回已启用、已发布、未删除且处于有效时间范围的数据。

页面模板继续作为后端数据契约和字段校验模型存在，但后端不实现 HTML、CSS、JavaScript、Vue 组件或页面渲染。

### 2.3 数据、文件与运维接口

1. 提供 MySQL 8 初始化 SQL、增量 SQL、索引和初始化权限数据。
2. 实现本地文件上传、类型与大小校验、文件元数据和公开访问地址。
3. 保留 MinIO 存储适配边界，不在未启用时增加分布式存储复杂度。
4. 提供健康检查、结构化日志、操作日志和登录日志。
5. 提供 local、dev、prod 配置基线、Nginx API 反向代理示例和部署说明。
6. 提供数据库备份、升级和回滚指导。

### 2.4 前端团队责任

以下内容明确由其他前端团队负责，不进入本项目开发任务和验收：

1. 官网主站、招生就业专题站和 CMS 管理端的全部页面与路由。
2. UI/UX、视觉稿、组件库、动效和交互设计。
3. PC、平板、手机响应式适配及浏览器兼容。
4. Vue、TypeScript、HTML、CSS、JavaScript 和前端构建工程。
5. 页面级 SEO 标签渲染、语义结构、可访问性和静态化或 SSR。
6. 前端自动化测试、页面性能优化、静态资源部署和前端线上故障处理。

后端团队负责解释接口、修复后端契约问题并提供联调支持，但不代替前端团队实现页面逻辑。

### 2.5 接口联调、测试与部署

1. 提供 OpenAPI、字段字典、统一错误码和必要的请求响应示例。
2. 执行后端单元测试、仓储测试、接口集成测试、权限测试和安全边界测试。
3. 使用前端团队确认的调用场景进行契约联调，接口变更通过版本记录管理。
4. 提供 MySQL 初始化脚本、Nginx API 配置、环境变量清单和后端部署说明。
5. 完成约定服务器上的后端服务、数据库连接、上传目录和 API 健康检查验证。
6. 前端站点域名、静态资源、页面上线和最终整站验收由前端或项目总包团队负责。

### 2.6 三年后端维护服务

维护期自后端验收之日起计算，包含：

1. 已交付后端接口缺陷修复。
2. 后端服务运行巡检、日志排查和故障定位。
3. 常规依赖安全更新和兼容性修复。
4. 数据库备份与恢复指导。
5. API 反向代理、证书续期和服务器迁移的后端技术协助。
6. 接口使用咨询和远程支持。

维护不包含前端页面问题、新增业务模块、页面改版、内容代运营、历史数据整理、驻场服务和第三方资源费用。超出范围的需求按变更流程单独评估。

## 3. 明确不建设的内容

以下内容不在后端专项范围内，也不作为当前架构的默认扩展任务：

1. 任何前端页面、组件、视觉、响应式、浏览器兼容和前端部署工作。
2. 复杂站群、院系多站点自助建站和多租户管理。
3. 多语言内容、人工翻译和多语种 SEO。
4. 短信、邮件、支付及相关第三方服务接入。
5. 复杂 BI、领导驾驶舱、用户画像和数据大屏。
6. 深度智慧校园集成，包括统一身份认证、数据中台、教务、学工、财务和门禁等业务系统对接。
7. 在线报名、在线缴费、咨询留言和招聘撮合等复杂互动业务。
8. 复杂多级审核、跨部门流程和电子签章。
9. 商业全文检索、视频转码、直播和大规模音视频分发。
10. 旧站全量数据迁移、历史数据清洗和内容代录入。
11. 等保测评、渗透测试认证和专项安全整改服务。

新增上述需求时，须形成变更单，重新评估费用、工期、架构和验收标准。

## 4. 技术架构

### 4.1 架构原则

1. 采用轻量化单体模块架构，一个 Java 服务承载后台 API 和公开 API。
2. 主站与专题站共用栏目、内容、媒体和权限模型，通过 `site_type` 区分数据。
3. 不引入微服务、服务注册中心、消息队列和分布式事务。
4. MySQL 作为必选持久化组件；Redis 仅在缓存需求明确时启用。
5. 文件存储默认使用服务器本地目录，可按部署环境切换 MinIO。
6. Nginx 只作为 API 与上传文件的反向代理示例，前端静态资源部署不在本项目范围内。
7. 页面采用固定模板库和预定义区块，模板配置只保存经过白名单校验的结构化数据。
8. 栏目、内容和页面区块共用站点隔离规则，禁止主站与专题站跨站点引用数据源。

### 4.2 技术基线

| 层级 | 技术选型 |
| --- | --- |
| 后端语言 | Java 25 |
| 构建工具 | Gradle 9.x，编译目标与运行时均为 Java 25 |
| 后端框架 | Spring Boot 4.1.0 |
| 安全框架 | Spring Security OAuth2 Resource Server + Spring JwtEncoder/JwtDecoder + BCrypt |
| 数据访问 | MyBatis Spring Boot Starter 4.0.1 / MyBatis |
| 数据库 | MySQL 8.x |
| 缓存 | Redis，可选启用 |
| 文件存储 | 本地文件存储，保留 MinIO 适配接口 |
| API 文档 | Springdoc OpenAPI 3.0.3 / Swagger UI |
| 接口调用方 | 外部 CMS、官网和专题站前端，不限定其技术栈 |
| 部署 | Java 25 服务 + MySQL 8 + Nginx API 反向代理 |

### 4.3 部署结构

```text
外部前端应用
   |
Nginx / API Gateway
├── /api          school-backend
└── /uploads      公开媒体文件

school-backend (Java 25)
├── Admin API
├── Portal API
├── Auth & Security
├── CMS Modules
└── Storage Adapter
   |
MySQL 8
└── Redis / MinIO（按部署需要启用）
```

## 5. 里程碑与工期

后端完整范围计划周期为 10 个自然周，已完成工作计入对应里程碑。接口清单、数据库环境、部署权限或前端联调用例延迟造成的等待时间不计入开发工期。

| 里程碑 | 周期 | 主要工作 | 阶段交付与验收门槛 |
| --- | --- | --- | --- |
| M0 范围与契约 | 第 1 周 | 后端边界、接口清单、角色、数据源和部署环境确认 | 后端范围、API 清单和责任矩阵确认 |
| M1 架构与数据 | 第 1-2 周 | Java 25 工程、数据库、安全、统一响应和 OpenAPI | 服务可启动，SQL 可初始化，认证权限生效 |
| M2 Admin API | 第 3-6 周 | 用户权限、栏目内容、模板区块、Banner、媒体、配置、SEO 和日志 | CMS 所需管理接口形成完整业务闭环 |
| M3 Portal API | 第 6-8 周 | 导航、页面聚合、公开内容、搜索、SEO 数据和文件访问 | 主站与专题站前端可按契约动态取数 |
| M4 联调与测试 | 第 9 周 | OpenAPI 校对、调用场景联调、权限、安全和回归测试 | 约定接口通过测试，阻断问题清零 |
| M5 后端交付 | 第 10 周 | MySQL 验证、部署演练、回滚、文档和交接 | API 服务可部署，交付资料齐全，进入后端维护期 |

M2 与 M3 可在公共数据契约稳定后部分并行。前端页面排期不进入上述 10 周工期，整站上线时间由前后端总排期另行确定。

## 6. 工作分解

### 6.1 需求与设计

1. 确认主站与专题站栏目结构、内容来源和管理员角色。
2. 确认 Admin API 与 Portal API 的调用场景、字段、状态和权限边界。
3. 冻结八类模板编码、适用栏目、字段矩阵和区块类型的数据契约。
4. 确认文件类型、大小、公开策略、存储路径和清理规则。
5. 确认错误码、分页、时间、枚举、空值和接口版本规则。
6. 形成后端需求基线，前端展示需求仅在影响接口契约时进入变更清单。

### 6.2 后端与数据库

1. 建立 Java 25 + Spring Boot 4.1.0 工程和环境配置。
2. 完成统一响应、异常、参数校验、分页和 OpenAPI。
3. 完成 MySQL 表、初始化数据和索引。
4. 使用 Spring Security OAuth2 Resource Server 和 Spring JwtEncoder/JwtDecoder 完成 JWT 登录、角色权限和操作审计。
5. 完成栏目、内容、附件、Banner、媒体、站点配置、友情链接和 SEO API。
6. 完成官网公开 API、搜索和浏览量能力。
7. 完成存储适配、上传校验和文件访问控制。
8. 增加栏目详情模板与模板配置、内容受控扩展数据和 `cms_page_section` 增量 SQL。
9. 完成模板注册表、编辑 schema、区块管理和公开页面聚合 API，并校验跨站点引用和 JSON 配置。

### 6.3 接口契约与联调

1. 维护 OpenAPI、权限编码、错误码、枚举和字段字典。
2. 为登录、分页、上传、内容发布、页面聚合和搜索提供示例请求响应。
3. 统一 ISO-8601 时间、分页参数、排序规则和逻辑删除数据可见性。
4. 与前端团队确认接口契约，记录兼容性变更并避免无版本破坏性修改。
5. 配置必要的 CORS、API 反向代理和上传文件访问规则。
6. 联调只处理接口、数据、鉴权和后端性能问题，不承担页面实现。

### 6.4 测试与安全

1. 执行领域服务、仓储映射、MyBatis Mapper 和控制器集成测试。
2. 覆盖 401、403、角色权限变化、账号禁用和关键防锁死规则。
3. 覆盖上传限制、文件访问、XSS 清洗、危险 URL、参数越界和跨站点引用。
4. 覆盖草稿、发布、下线、定时生效、逻辑删除和内部引用生命周期。
5. 覆盖 Portal API 空数据、分页边界、SEO 回退和过期 Banner 过滤。
6. 在可用 MySQL 8 环境执行初始化、增量升级和关键 CRUD 验证。

### 6.5 部署与交付

1. 完成 Java 25 服务构建、环境变量、日志目录和上传目录配置。
2. 完成后端部署、数据库备份、升级和回滚演练。
3. 验证 Nginx API 反向代理、上传文件访问和健康检查。
4. 提交源代码、初始化及增量 SQL、OpenAPI、模板数据契约、部署文档和测试报告。
5. 提供后端运维与接口使用交接，不提供前端用户操作手册。

## 7. 商务口径

原 42.8 万元标准整包和约 38.8 万元精简整包均包含前端设计与开发，现已不适用于后端专项。后端报价须基于以下工作包重新核定，本计划不直接沿用或按比例拆分原整包金额：

| 后端工作包 | 核价范围 |
| --- | --- |
| 需求与接口设计 | 后端范围、数据模型、权限、模板数据契约和 OpenAPI |
| Admin API | 认证权限、CMS 内容运营、媒体、配置、SEO 和日志接口 |
| Portal API | 主站及专题站公开聚合、搜索、SEO 数据和文件访问接口 |
| 数据与存储 | MySQL SQL、索引、升级脚本、本地存储及 MinIO 适配边界 |
| 质量与安全 | 自动化测试、权限边界、上传安全、XSS 和跨站点校验 |
| 部署与联调 | 后端部署、API 代理、数据库验证、文档和前端接口联调 |
| 三年后端维护 | 后端缺陷、依赖升级、日志排查和数据库运维协助 |

建议付款节点仍可按 `30% 启动 + 30% Admin API + 30% Portal API 与后端验收 + 10% 交接维护` 设置，具体金额和触发条件以新的后端合同为准。

## 8. 验收标准

### 8.1 通用接口

1. 后端使用 Java 25 编译运行，统一响应、分页、参数校验和错误码符合约定。
2. OpenAPI 可访问，已交付接口、DTO、枚举、权限和状态码与实现一致。
3. 时间使用 ISO-8601，分页、排序、空值和逻辑删除规则保持一致。
4. 不存在资源、数据冲突、未登录和无权限分别返回约定的 404、409、401 和 403。

### 8.2 Admin API

1. 认证、用户、角色和权限接口可完成既定管理流程并执行权限校验。
2. 栏目、内容、附件、模板、页面区块、Banner、媒体、站点配置、友情链接和 SEO 接口可用。
3. 内容草稿、发布、下线、置顶、推荐和附件替换状态流转正确。
4. 跨站点引用、被占用资源删除、危险链接和非法 JSON 配置被明确拦截。
5. 关键写操作和登录行为可通过日志接口追溯。

### 8.3 Portal API

1. 主站与招生就业专题站的数据按 `site_type` 隔离。
2. 公开接口只返回已启用、已发布、未删除且当前有效的数据。
3. 导航、Banner、页面区块、栏目列表、内容详情、友情链接和站点配置可按契约查询。
4. 搜索分页、发布时间过滤、SEO 回退、附件信息和浏览量更新符合约定。
5. 空数据返回稳定结构，不依赖前端解析数据库 JSON 或推断模板规则。

### 8.4 数据、文件与安全

1. 初始化及增量 SQL 可在 MySQL 8 执行，关键表、索引和初始化权限正确。
2. 文件上传执行大小、扩展名、MIME 类型、路径和非执行目录限制。
3. 富文本清洗脚本、事件属性和危险协议，外部 URL 只允许约定协议。
4. JWT 签名、签发方、有效期、账号状态和实时权限校验通过测试。
5. local 与 MyBatis 实现遵循同一业务语义，正式环境关键 CRUD 完成联库验证。

### 8.5 交付物

1. 提交后端源代码、Gradle 配置、初始化及增量 SQL。
2. 提交 OpenAPI、字段与枚举说明、环境变量和错误码说明。
3. 提交自动化测试结果、部署文档、Nginx API 示例、备份和回滚说明。
4. 后端服务、数据库连接、上传目录和健康检查在约定环境运行正常。
5. 页面可访问性、视觉、响应式、浏览器兼容和前端性能不作为后端验收项。

## 9. 维护服务等级

维护服务默认为工作日远程支持，法定节假日和 7×24 值守需另行约定。

| 级别 | 示例 | 首次响应目标 | 处理方式 |
| --- | --- | --- | --- |
| P1 | API 服务整体不可用、认证服务不可用 | 2 个工作小时 | 优先恢复后端服务，并持续同步处理进度 |
| P2 | 内容发布、查询、上传等核心接口异常 | 4 个工作小时 | 1 个工作日内给出修复或临时方案 |
| P3 | 非核心接口缺陷、接口使用咨询 | 1 个工作日 | 纳入最近维护窗口处理 |

前端页面缺陷由前端团队处理。因云平台、运营商、前端部署、第三方服务、校方自行修改环境或不可抗力导致的问题，后端团队负责协助定位，恢复时间不作为后端维护承诺。

## 10. 项目协作与变更

1. 业务方指定一名负责人统一确认栏目、角色、接口场景和验收意见。
2. 前端团队指定接口负责人，统一反馈字段、鉴权、跨域和联调问题。
3. 业务方按里程碑提供栏目、数据样例、文件样例、数据库及服务器权限。
4. OpenAPI 和字段字典作为联调基线；确认后的破坏性修改须记录版本和影响。
5. 变更单需说明需求、影响接口、数据迁移、费用、工期和验收方式。
6. 前端实现或排期变化不自动转化为后端任务，只有契约变化才进入后端评估。

## 11. 主要风险

| 风险 | 影响 | 应对策略 |
| --- | --- | --- |
| 范围持续扩大 | 报价和工期失控 | 以后端接口清单为基线，范围外需求走变更单 |
| 接口场景确认延迟 | Portal API 和联调延期 | M0 冻结调用场景、样例数据和责任人 |
| 前端频繁改变字段需求 | 接口反复修改并产生兼容问题 | 以 OpenAPI 为契约，破坏性修改走版本和变更评估 |
| 页面 SEO 期望转嫁后端 | 责任边界不清 | 后端提供 SEO 数据，标签渲染、SSR 或静态化由前端负责 |
| 上传内容存在安全风险 | XSS 或恶意文件 | 富文本过滤、扩展名与大小限制、非执行目录存储 |
| 本地文件不利于迁移 | 环境切换成本增加 | 使用 StorageService 隔离存储实现，保留 MinIO 适配 |
| MySQL 联库环境缺失 | 正式仓储行为无法完整验证 | 在 M4 前提供 MySQL 8 测试环境和执行权限 |
| 模板数量持续增加 | DTO、校验和公开聚合复杂度上升 | M0 冻结八类模板，新增结构走变更单 |
| JSON 扩展字段失控 | 数据不可检索、不可校验 | 只保存少量模板专属字段，按模板 schema 校验并保留公共字段列 |
| 模板数据源悬空或跨站点 | 前端取得错误或空数据 | 保存时校验引用，目标禁用删除前检查依赖，Portal API 再过滤 |
