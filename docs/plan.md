# 高校官网建设项目开发计划

## 1. 执行原则

本计划基于 README.md 中的高校官网建设项目规划制定，首版以 MVP 可交付为目标，优先完成后台可运营、前台可展示、接口可联调、系统可部署的闭环。

首版开发范围：

```text
后台登录
用户角色权限
栏目管理
内容管理
Banner 管理
媒体库
站点配置
友情链接
官网首页
栏目列表页
内容详情页
招生就业专题站展示
搜索
部署脚本与文档
```

暂缓到二期：

```text
在线报名
留言咨询
多级审核流
统一身份认证
旧站数据迁移
多语言
复杂全文检索
视频转码
等保整改专项
```

## 2. 里程碑计划

### M1 项目基础搭建

目标：完成后端基础工程、基础配置和公共能力骨架。

任务：

1. 调整 Gradle 工程为 Spring Boot 3.x + Java 17 项目。
2. 建立基础包结构：common、config、security、modules、infrastructure。
3. 添加 Spring Web、Validation、Security、MyBatis、MySQL、Redis、JWT、OpenAPI 依赖。
4. 提供 application.yml 示例配置。
5. 实现统一返回对象 ApiResult。
6. 实现分页返回对象 PageResult。
7. 实现错误码枚举 ErrorCode。
8. 实现全局异常处理 GlobalExceptionHandler。
9. 建立数据库初始化目录 src/main/resources/db。

验收标准：

```text
后端服务可以启动
基础健康检查接口可访问
接口返回格式统一
参数校验和异常处理生效
OpenAPI 文档入口可访问
```

### M2 数据库与基础权限

目标：完成系统基础表结构和后台认证权限闭环。

任务：

1. 编写 MySQL 8 初始化 SQL。
2. 创建 sys_user、sys_role、sys_permission、sys_user_role、sys_role_permission 表。
3. 创建 sys_login_log、sys_operation_log 表。
4. 初始化超级管理员、基础角色和菜单权限。
5. 实现 BCrypt 密码加密。
6. 实现 JWT 登录认证。
7. 实现当前用户信息查询。
8. 实现后台接口鉴权。
9. 实现未登录 401、无权限 403。

验收标准：

```text
初始化 SQL 可重复执行或有清晰执行说明
管理员可以登录后台 API
登录成功返回 JWT
当前用户接口返回用户、角色、权限信息
受保护接口未登录不可访问
```

## 3. 推荐开发顺序

1. 先完成 M1，保证后端工程可运行。
2. 再完成 M2，建立登录、鉴权和数据库基础。
3. 接着完成 M4 的栏目、内容、Banner、媒体库，因为它们决定前台展示数据模型。
4. 随后完成 M5，形成前台公开 API。
5. 再开发 M6 和 M7，后台管理端与官网前台可以并行。
6. 最后完成 M8，集中处理联调、部署、测试和文档。

## 5. 风险与处理

| 风险 | 影响 | 处理策略 |
| --- | --- | --- |
| 首版范围过大 | 延误 MVP 交付 | 严格按 MVP 范围推进，二期能力只预留接口和表字段 |
| SEO 要求提升 | Vue SPA 不利于收录 | 首版支持基础 SEO 字段，后续升级 SSR 或静态化 |
| 文件存储迁移 | 本地存储不适合多实例 | 先抽象 StorageService，后续切换 MinIO |
| 权限粒度不清 | 后台菜单和按钮控制混乱 | 权限表同时支持菜单和按钮类型 |
| 招生就业模型重复 | 表结构冗余 | 首版通过 site_type 复用栏目和内容模型 |
| 富文本安全 | 前台 XSS 风险 | 后端或前端展示前进行 HTML 安全过滤 |

## 6. 完成定义

首版完成需同时满足：

1. 后端服务可启动，核心 API 可通过 OpenAPI 或接口工具验证。
2. 后台管理员可登录并维护栏目、内容、Banner、媒体库和站点配置。
3. 官网前台可动态展示后台发布内容。
4. 招生就业专题站可独立展示内容。
5. 搜索、友情链接、浏览量、文件访问等基础能力可用。
6. 数据库初始化、部署说明、接口说明和测试说明齐备。
7. 核心流程通过功能测试、权限测试和文件上传测试。
